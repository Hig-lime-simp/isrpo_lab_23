## Описание проекта

В ходе лабораторной работы было создано интерактивное веб-приложение "Крестики-нолики" (Tic-Tac-Toe) — классическая игра для двух игроков. Проект реализован с использованием чистого JavaScript, HTML5 и CSS3. Применён объектно-ориентированный подход с выделением логики игры в отдельный класс и использованием ES6 модулей для лучшей организации кода.

---

## Правила игры

- Игровое поле представляет собой квадрат 3×3 клетки
- Первый игрок ходит крестиками (X), второй — ноликами (O)
- Игроки по очереди ставят свой символ в свободные клетки
- Цель игры — первым выстроить连续 линию из трёх своих символов:
  - по горизонтали
  - по вертикали
  - по диагонали
- Если все клетки заполнены, но победитель не выявлен — объявляется ничья
- После завершения партии можно начать новую игру кнопкой "Начать заново"

## Функциональность

### Игровая механика
- **Клик по клетке** — установка символа текущего игрока
- **Автоматическое определение победителя** — проверка всех выигрышных комбинаций после каждого хода
- **Определение ничьей** — проверка заполненности всех клеток
- **Переключение игроков** — после каждого хода
- **Блокировка ходов** — после победы или ничьей клетки становятся неактивными
- **Перезапуск игры** — кнопка "Начать заново" сбрасывает состояние

### Интерфейс
- Отображение текущего игрока в статусной строке
- Визуальное выделение клеток при наведении
- Сообщение о победителе или ничьей
- Адаптивное центрирование игрового поля

## Логика работы

### Как определяется победитель?

В классе определён массив `winConditions`, содержащий все возможные выигрышные комбинации (индексы клеток):

```javascript
this.winConditions = [
    [0, 1, 2], // верхняя горизонталь
    [3, 4, 5], // средняя горизонталь
    [6, 7, 8], // нижняя горизонталь
    [0, 3, 6], // левая вертикаль
    [1, 4, 7], // центральная вертикаль
    [2, 5, 8], // правая вертикаль
    [0, 4, 8], // диагональ 
    [2, 4, 6]  // диагональ 
];
```

После каждого хода в checkResult() происходит проверка:

```javascript
for (let i = 0; i < this.winConditions.length; i++) {
    const [a, b, c] = this.winConditions[i];
    if (this.board[a] && 
        this.board[a] === this.board[b] && 
        this.board[a] === this.board[c]) {
        // Найдена победная комбинация!
    }
}
```
## Как обрабатываются клики?

1. Получение целевой клетки и её индекса через data-index

2. Проверка: свободна ли клетка и активна ли игра

3. Обновление массива board и визуальное отображение символа

4. Вызов checkResult() для анализа ситуации

5. Если игра продолжается — смена игрока

## Как работает перезапуск?

Метод ```restartGame()```:

- Сбрасывает currentPlayer на "X"

- Очищает массив board

- Устанавливает gameActive = true

- Обновляет статусную строку

- Очищает текст во всех клетках

## Структура данных

Массив board
```javascript
this.board = ["", "", "", "", "", "", "", "", ""];
```
Индексы 0–8 соответствуют клеткам игрового поля (слева направо, сверху вниз)

Пустая строка "" означает свободную клетку

Символы "X" или "O" означают занятую клетку

Состояние игры
```javascript
gameActive: true — игра продолжается

gameActive: false — игра завершена (победа или ничья)
```

## Алгоритмы

#### Алгоритм проверки победы

1. Получить все выигрышные комбинации из winConditions
2. Для каждой комбинации [a, b, c]:
   - Проверить, что клетка a не пуста
   - Проверить, что клетки a, b, c содержат одинаковые символы
3. Если хотя бы одна комбинация подходит — объявить победу

#### Алгоритм определения ничьей
1. Проверить массив board на наличие пустых строк ("")
2. Если пустых клеток нет и победитель не определён — объявить ничью

#### Алгоритм смены игрока
```javascript
currentPlayer = currentPlayer === "X" ? "O" : "X";
```
Тернарный оператор переключает между "X" и "O" после каждого успешного хода.
